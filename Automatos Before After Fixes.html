Fix #5: Align Badge Semantic Colors
Medium Impact
30 min

📍 What's Wrong

Status badges use inconsistent color tokens across pages. Some use --success, others use --primary for the same semantic meaning (active, online, healthy).

  • Active/Online: Should always use hsl(var(--success)) (green)
  • Warning/Attention: Should use hsl(var(--warning)) (amber)
  • Error/Failed: Should use hsl(var(--destructive)) (red)
  • Neutral/Idle: Should use hsl(var(--muted-foreground)) (gray)
❌ Before (Mixed)
Active
Processing
Running
Failed

• "Active" and "Running" use orange (primary) • Should be green (success)

✅ After (Semantic)
Active
Processing
Running
Failed

✓ All "active" states = green ✓ Semantic consistency ✓ Instant recognition

// Create a helper function for semantic status colors function getStatusVariant(status: string) { switch (status) { case 'active': case 'running': case 'online': case 'healthy': return 'success' case 'warning': case 'attention': case 'pending': return 'warning' case 'error': case 'failed': case 'offline': return 'destructive' default: return 'secondary' } } // Then use it: <Badge variant={getStatusVariant(agent.status)}> {agent.status} </Badge>
Impact: Users recognize status at a glance, color = meaning across entire platform
