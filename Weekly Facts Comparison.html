Left = render with the current weekly-facts.md (Title-Case headlines, long words, fixed font sizes). Right = render with the fixed data + auto-sized headline. Every render below is the actual render/index.html the renderer uses, served at IG-post 1080×1350 and scaled to fit.

The two fixes

  • Headline grammar: 2–4 ALL-CAPS lines split by |, ≤ ~10 chars per line. Optionally suffix a line with @brick to brick-color it. Title-Case multi-syllable words (Workspace, Coordination) overflow at display weight.
  • Auto-sizer in render/index.fixed.html: if no fontSize URL param, the renderer computes one from the longest line × line count so the headline always fits ~85% width × ~45% height — capped at the per-size baseline.
  • Card body length: aim ≤ 160 chars per card body, ≤ 28 chars per heading. Stat values ≤ 4 chars (10+, 95%, $2B).
